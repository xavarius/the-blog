I made a product with AI help. A Journal.

My endeavour as a software engineer budling a digital product with the little help from my friends: ChatGpt, Windsurf, Claude.

ThI figured how to help myself and my wife with groceries done regularly for our household. I got extremally exited, sat and started coding. Did some research with chatgpt at first, then was jumping between chatgpt windsurf chrome for 3h straight. 

The second day

It’s saturday. I’m playing with kids and doing household chores but I am manacally thinking about stuff that I want to do, how much I would eventually earn (that’s totally day dreaming) and how much happy I will be when it’s done. On the other hand I’m scoping down the mvp — I want to make this product work for my family and observe it for some time — before the (beta) market release.

The third day since starting

Windsurf
I did a bit of high level code review of the codebase provided by Windsurf. I had to ask it to analyse dead code occurences — it went via all files and then proposed couple of eslint commands to check for warnings, loose ends, not used method and dependencies. That’s quite all right. The codebase is a bit cleaner now.

Although, the codebase is still bloated with unnecessary things which mean — I takes more time to validate what AI is delivering — but I won’t give up. 

I started thinking about “refactoring”. My initial idea is to learn AI based refactoring (if this is possible) file by file. This is not optimal — but let’s try. I also want to, as a baseline, to do UI testing — I need to be able to verify funcitonality of the system quicku — I need a feedback loop — AI is to random to depend on its results — and more and more complex the system will become — then AI will fail more (like human?). Yeah, let’s see. 

I’m also consantly working on removing complexity from the system — as above — I feel like senior sofgtware engineer — I’m removing code all the time instead of writting it! xD 

I also see how much additional stuff without asking me Windsurf generates. That’s good since it’s in example implement target price component everywhere it makes sense, on the other hand sometimes it’s a total redudancy/overhead. Interesting. Let’s try to coach it to ask questions before implementation (like humans? :D ).

This is funny. 

Ok, now. It’s stuck for the third time with “refactor” 

Maciej out! I need to code now!

Fifth day

I figured I’m somehow not effective anymore. I’m thinking about Price Checker product all the time but I’m not getting things done.

I started again, from researching tech stack and ideas with ChatGpt. Since Windsurf started my project on a deprecated template and I was able to remove much of the code created by AI — and app is still running I figured I need to do couple of things, which are:

  •  Analyse the idea of the app
  • Create a fundational yet mininal tech stack for this app
  • Instruct Windsurf stricly around software engineering practices, including software architecure, usage of linters, unit testing, automation testing and usage of version control
  • I need to more in the loop and enforce good engineering practises — in example I will create tests per each step with AI — seems like AI is breaking stuff all the time and I need to keep the changes as minimal as possible and as precise as possible. I can’t keep a snapshot of a project in mind — but I can define specification vs verification rules- TEST! In this case, as in any other software project, I will minimalise regressions.

So, I made a migration from clean React into Next.js approach.

What’s interesting the UI basics stayed BUT the presentation changed a lot. New features emerged, like About section and Github cta. I don’t like this — AI changed more then I anticipated and I lost control over changes. Also, it copy pasted a lot of dirs and files from the old project. The remedy was simple — I asked AI to go via project and define which dirs and files are really used. 

The migration is done. 

I still lack a partnership. So I started teaching AI to do basic stuff. I have created .windsurf file which teaches AI guidelines. I provided basic TS guidelines (removed one about JSDocs) and also provided guidelines around making commits to Git. I like the response and how AI works with git. I asked it also to start its messages with “[AI]” tag — this way I will be able to differenciate between my commits (yes, I still do code this app) and AI commits. I also will be able to create metrics around the code, AI vs me. 

For now I have defined the following steps for myself:

  1. App tour with AI
  2. Ensure project files and dependencies are up to date — including removing obsolete stuff
  3. Create lin
  4. ter workflow — es lint
  5. Create a workflow with AI (windsurf rules + memory + examples?) to create TDD flow (or at least to implement unit tests per given code)
  6. Fix Next.js warning and errors
  7. Let’s remove features AI created and which are not needed (github, about, category field). 
  8. Finally, test if AI can code, commit and test in one go. I will manually verify the app, for now. 

It’s 4pm, summary time

  • migrated from bare react to next.js
  • Created workflow for the AI to analyse, code, eslint+prettier, run unit tests and then commit code imitating my style of coding
  • While doing the above I can clearly see AI uses “old” frameworks and libs by default, I suppose it’s because more examples are there. So, after the initial setup AI and I had to update almost everything again, from Next.js 14 into 16, to EsLint configuration etc. Shame.
  • Inherited unwanted features which I will have to delete tomorrow

Day 6th

I started fom manual migration of Eslint since Claude 3.7 wasn’t able to deliver it. Then I had to manually install dependencies.

npm install @eslint/compat globals @eslint/js @eslint/eslintrc -D

I also failed with instructing Windsurft to do the standard engineering workflow: analyse, code, linter, tests, commit. It wasn’t able to connect the dots and use appropriate commands with appripriate data.

I figured, since at least lint, test, commit flow is deterministic I will instruct Windsurf to create a script which does the above. The script accepts a commit message (string) as an input, then runs linter (with — fix option) and prints summary. Then runs tests and prints summary. Then adds all files to staging and commits them with nice looking commit message.

Now, I forced Windsurf to do work autonymously. I request feature, it codes, verifies, fixes and commits. Perfect! 

I kind of feel I know how to work with AI now. The current AI. It will change. That feels good. I can focus on the product now — since the setup looks good enough. As software engineer I wanna a stable tech stack (since I know how much it’s important when scaling the solution i.ex.) but I also wanna to focus on the business core features. I feel I can now. This feels very, very good. 

I also thought I want to see insights around development with AI on a daily basis. I want to track impact of AI vs myself. And I really want to understand what’s the code complexity degree in the project, and how it changes.

I made a script with Windsurf to print some of the insights.

I need to work on the code complexity metric, thou. 

I dropped the code complexity topic. I might not be relatable in the age of AI. Anyway, I have each and every git commit annotated with AI or human contribution indicator — meaning, I can create insights retrospectively anytime I want. That’s powerfull. 

On the other hand, I’m again distracted from the core business logic with fancy shawncy scrippting around cool topics. 

Creating every toy in the developer toolbox I dreamt about seems so easy -but in the end I’m not doing the CORE work. That’s a trap. That is tricky. 

The Seventh Day

I started to think that code complexity matters. Ok, we’re in the age of AI — but there is no outcome without strict human supervision. This means code complexity matters since a person have to oversight, read and maintane a codebase. I need to incorporate this metric into my day to day insights. 

I must say that AI is holding me back with new features. Sure, I could instruct it to create code and voila! It’s done. Kinda. Or not done. But I want to be sure that AI can create a deterministic outcomes according to ruleset I created in the long term. And this is a tricky part. I spend a lot of time tutoring AI how to work. It also constantly trying to incorporate deprecated tools, mechanisms, libs and so on. It is so annoying! 

I see that Windsurf starts to act wierd with things it was able to do, like committing with the usage of the script we build together. I asked to validate “the Windsurf memory” and remove redundant and overlapping memory entries — with must have step to confirm all changes with me — to ensure those rules have sense. 

The 12th day

Windsurf chat broke. I had to start a new one. The old one wasn’t able to execute any command anymore. Thankfully, cascade memory stayed and I had windsurf rules defined per project.
